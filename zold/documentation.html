1. Abstract

flat.dbrqx.com is a one page site, a landing page. This page is for selling one particular flat. It is not supposed to sell many flats like a ecommerce site. Something similar to this page: http://www.fotocasa.es/building/building_detail.aspx?ai=138101509&opi=10&cu=es-es&rdm=57 but it will have far less content. The plan is to use lazy loading and ratina images too.

2. Contents of website

The website should have:

  • A logo. For example, logo could be a house shape like an house icon. Or a building shape. Logo should never be copied. And must be original. If the logo is vectorish(A clip art or line drawing) then it must be in svg
  • The website in future will be made multilanguage. So an option to select language should be there. Global Rule
  • The website will start off with the text For Sale
  • Next will come a slideshow. Here different images of the flat will be shown. Just below this large slideshow there should show a carousel showing multi images in one slide(thumbnails). Exactly like Flex Slider. Although a new original style is more better, for example The Owl Carousel with thmbnails. Nearly 20 images would be there in bottom carousels(thumbnails). The bottom carousel's each slide could have nearly 4 or 5 images.
  • When hovered over the top slider's main image two things will show, 1. A description of the image, and, 2. two images -- first image will be phone number and second will be email id. The phone number and email id's are intentionally made images so as to avoid future spam of bots. The description must be big(font-size). What would happen to the description in touch devices like tab and even on small screen potrait mobile is to be decided by the designer. A solution on mobile devices is required.
  • After the slideshow at the bottom we'd have two links. These links would go to other pages. These are used for google crawler and seo position.

3. Global Rules

  1. The page height must be less. Nearly one page height is preferable so that when user scrolls one or two times he reaches the footer. For flat.dbrqx this is automatically fullfiled as we have low content.
  2. Everthing will work on hover. That is a tab, menu or slider will run with mouseenter event. On mobile this will work on click(tap) event. Any click is supposed to load another page. The design must be made with this fact kept in mind.
  3. All of our websites will be multilingual. So there must be some way a visitor can select the language. Like a dropdown in header, for example.
